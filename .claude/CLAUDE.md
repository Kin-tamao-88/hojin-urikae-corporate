# CLAUDE.md

## LP制作 共通ルール

### 基本方針

-   Tailwind CSSはモバイルファーストで実装する。
-   SPレイアウトを基準に設計し、PCは `md:` / `lg:` で拡張する。
-   PC完成後にSPを後付けしない。
-   承認済みのコピー・構成・原稿は変更しない。
-   PCデザインは、明示的な指示がない限り変更しない。
-   他案件のコピー・デザイン・配色を混ぜない。

### コーディングルール

-   `style={{}}` の inline style は原則使用禁止。
-   Tailwind Utility Class を優先する。
-   `!important` は使用しない。
-   固定高さ・固定幅は必要最小限とし、SPでは `h-auto` `w-full`
    を基本とする。
-   Heroなどで absolute を使う場合は `md:` 以上で適用する。

悪い例

``` tsx
<div style={{ height: '560px', position: 'absolute' }}>
```

良い例

``` tsx
<div className="relative h-auto md:absolute md:h-[560px]">
```

### レスポンシブ

-   SPとPCを同時に考慮して実装する。
-   SPは読みやすさを優先し、縦積みを基本とする。
-   改行は必要に応じて `<br />`
    を使用し、PC/SPで意図した見え方を維持する。
-   レスポンシブ対応後はPC表示も必ず再確認する。

### デザイン

-   AIっぽいデザインにしない。
-   SaaS感・管理画面感を避ける。
-   カードの乱用を避ける。
-   情報の階層を明確にし、本文が小さく・薄くならないよう注意する。
-   余白は広げすぎず、情報密度を維持する。

### 作業フロー

1.  GPT完成イメージを確認
2.  Claude Codeで実装
3.  PC確認
4.  SP確認
5.  Consoleエラー確認
6.  必要な修正のみ実施

### 禁止事項

-   原稿を勝手に変更しない。
-   コピーを改善目的で書き換えない。
-   新しい装飾やカードを追加しない。
-   推測でレイアウトを変更しない。
